Restore generic MBR code so that Windows boots again - Rescapp - Documentation
  • Step 1

    Rescapp searchs for primary partitions on your disk.
  • Step 2

    If such partitions have been found Rescapp presents you the partitions. At the same time you are prompted:

    Which partition?

    You are supposed to choose your Windows (probably you only will see one) and press the OK button. If you have several Windows the best Windows to choose is the first one that you installed on your computer.

  • Step 3

    Rescapp searchs for hard disks present on your system based on information found on /proc/partitions file.
  • Step 4

    You are prompted:

    Which hard disk to install Windows mbr on?

    You are supposed to take a look at hard disk sizes (to be able to identify your hard disks) and choose (usually) the one that it is set to be first one to boot at your BIOS settings.

  • Step 5

    You are prompted:

    Which is the right position for this hard disk?

    Just select the right boot order position for the prompted hard disk according to your BIOS settings.

  • Step 6

    Rescapp with all gathered information is going to set up a Windows mbr in your choosen hard disk so that the partition you have choose in booted at your machine boot.Please wait. Currently (Rescatux 0.23) this step is time consuming but no Please wait message has been implemented.
  • Step 7

    You are informed about the mbr installation process.

    Message is self explanatory. You either get:

    Success! Windows mbr was installed OK! :)

    or

    Failure! Windows mbr was not installed. Something went wrong! :(

  • Step 8

    A log file called winmbr.log.txt can be found at Desktop logs folder.
