Update GRUB Configuration File - Rescapp - Documentation

  • Step 1

    Rescapp searchs for partitions that have on them the following file: /etc/issue based on information found in /proc/partitions file.
  • Step 2

    If such partitions have been found Rescapp presents you partitions that have on them the following file: /etc/issue. At the same time you are prompted:

    Which partition?

    You are supposed to choose your main GNU/Linux distribution (probably you only will see one) and press the OK button.

  • Step 3

    Rescapp searchs for hard disks present on your system based on information found on /proc/partitions file.
  • Step 4

    You are prompted:

    Order hard disks according to boot order. Press OK to continue.

    . Take a look at hard disks and their sizes. Order the according to BIOS boot order and then press OK.

  • Step 5

    Rescapp with all gathered information is going to create a temporal device.map at your Linux installation (which will be deleted later) and try to update your grub configuration file on your selected hard disk thanks to your own distribution update-grub command.Please wait. Currently (Rescatux 0.23) this step is time consuming but no Please wait message has been implemented.
  • Step 6

    You are informed about the grub configuration file update process.

    Message is self explanatory. You either get:

    Success! Grub configuration was updated ok! :)

    or

    Failure! Grub configuration update went wrong! :(

  • Step 7

    A log file called update-grub.log.txt can be found at Desktop logs folder.
